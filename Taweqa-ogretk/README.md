## المميزات

- **عداد ذكي**: حساب الأجرة بناءً على المسافة، الوقت، والانتظار — مع عدادين مستقلين
- **خرائط OpenStreetMap**: تتبع GPS دقيق مع Leaflet و OSRM
- **نظام طلب رحلة بالخريطة**: الراكب يحدد نقاط المسار (حتى 20 نقطة) — أول نقطة انطلاق (أخضر) وباقي النقاط توقف (أزرق)
- **تقدير الأجرة المبدئي**: بناءً على المسافة بين النقاط ونوع الرحلة
- **ملاحظة للسائق**: الراكب يضيف ملاحظة نصية مع الطلب
- **عرض نقاط المسار للسائق**: شاشة تفاعلية تعرض نقاط الرحلة كاملة
- **إدارة الطلبات**: إرسال/استقبال/قبول/رفض طلبات الرحلات مع توزيع ذكي على السائقين
- **محادثة فورية**: شات مباشر بين السائق والراكب
- **محفظة إلكترونية**: شحن، سحب، اشتراك شهري
- **تسوية الرحلات**: ضبط سعر الكيلو، فتحة العداد، وقت الانتظار — مع اختيار طريقة الدفع (نقدي/محفظة)
- **نظام تقييم**: تقييم الرحلات بالنجوم مع تعليق
- **تتبع الرحلات**: مشاركة كود 6 أرقام للتتبع المباشر
- **نظام إحالة**: كود إحالة لكل مستخدم
- **PWA**: تطبيق ويب تقدمي قابل للتثبيت
- **CSP أمني**: سياسة أمان محتوى صارمة

## متطلبات التشغيل

1. حساب Supabase (مجاني)
2. رفع الـ SQL migrations في `Taweqa-ogretk/supabase/migrations/`
3. نشر Edge Functions: `trip-events`, `track-trip`, `paymob-charge`
4. إعداد Supabase Secrets:
   - `PAYMOB_API_KEY` (لخدمة Paymob)
   - `PAYMOB_INTEGRATION_ID`
   - `SITE_URL` (رابط التطبيق)
5. رفع الملفات على أي مستضيف يدعم الملفات الثابتة (Vercel، Netlify، GitHub Pages، إلخ)

## التطوير

```
git clone <repo>
cd Taweqa-ogretk
# حرر الملفات مباشرة — لا توجد أدوات بناء
```

افتح `index.html` في المتصفح أو استخدم `npx serve .` لخادم محلي.

## النشر

1. ارفع جميع الملفات إلى مستضيف ثابت
2. اضبط `SITE_URL` في Supabase Dashboard → Authentication → Settings
3. نشر Edge Functions عبر Supabase CLI أو Dashboard
4. تأكد من تطبيق جميع SQL migrations
